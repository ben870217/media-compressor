# MediaCompressor 優化規格 v1

狀態：已確認，作為後續優化與實作依據。

目的：整理本次預計優化的行為規格，讓後續開發、驗證與回顧都能對照同一份基準。

## 範圍

- 影片與照片都要支援批次處理。
- 影片與照片維持分開加入、分開處理。
- 需要支援原始比例偵測、比例調整與比例維持。
- 需要支援共用批次設定與單檔覆寫。
- 需要支援批次佇列、下載、歷程與失敗重試。

## 核心規格

### 1. 批次處理

- 單一批次最多 50 個檔案。
- 同一批次可重複加入相同來源檔案，但要標示為重複，不自動去重。
- 檔案順序可拖曳重排，預設維持加入順序。
- 佇列採序列處理，一次只處理一個檔案。
- 目前處理中的檔案設定鎖定，尚未處理的檔案可調整。
- 批次設定變更只影響尚未處理的檔案，不會回寫已完成結果。

### 2. 設定模型

- 保留「批次共用設定」與「單檔覆寫」。
- 單檔只覆寫被修改的欄位，不會覆蓋其他欄位。
- 批次設定變更不會自動清除既有單檔覆寫。
- 檔案可重設單一欄位，或重設整個檔案回到批次預設。
- 介面需要明確標示哪些欄位已套用單檔覆寫。

### 3. 來源比例偵測

- 影片與照片都要先偵測原始比例。
- 影片需考慮旋轉 metadata 與實際顯示方向。
- 偵測結果應顯示原始尺寸與標準化比例。
- 常見比例可歸類為：
  - `original`
  - `16:9`
  - `9:16`
  - `1:1`
  - `4:3`
  - `3:4`
  - `4:5`
  - `custom`
- `original` 代表維持每個檔案自己的原始比例。

### 4. 輸出比例與裁切

- 不允許拉伸變形。
- 提供三種比例處理模式：
  - 維持原始比例
  - 裁切填滿
  - 完整顯示並加留白
- 裁切填滿預設使用置中裁切。
- 可提供單檔焦點點位覆寫，讓裁切中心可調。
- 留白模式可選背景色，v1 不支援透明背景。

### 5. 輸出尺寸

- 尺寸設定視為長邊上限或品質上限。
- 輸出另一邊依比例自動計算。
- 來源解析度較低時不得自動放大。
- 影片輸出尺寸需維持偶數。
- 例：
  - `1080p + 1:1 => 1080 x 1080`
  - `1080p + 9:16 => 約 608 x 1080`

### 6. 影片規格

- 保留現有影片轉碼能力。
- 影片支援輸入與輸出前先做 codec / 尺寸相容性預檢。
- 若裝置不支援目標編碼能力，需阻擋對應檔案並提示可用替代方案。
- 目標容量是以單一檔案為單位，不是整批總容量。
- 若輸出超過目標容量，允許有限次重試。
- 重試次數上限為 2 次。
- 超過容差後保留最接近的結果並提示。

### 7. 照片規格

- 照片輸出維持目前靜態圖片流程。
- 目前僅支援靜態 JPEG、PNG、WebP，不支援 GIF 或其他動畫圖格式。
- 偵測到動畫圖檔時需明確拒絕，不可默默截第一幀。
- 照片輸出尺寸提供常用上限與自訂上限。
- 來源太小時不得自動放大，必要時改用品質或尺寸下修策略。
- 若仍無法達標，保留最佳結果並提示，不視為成功達標。

### 8. 背景與留白

- 影片預設背景為黑色。
- 照片預設背景為白色。
- 可選背景模式：
  - 黑色
  - 白色
  - 模糊延展
  - 自訂純色

### 9. 下載流程

- 保留單檔預覽、重新命名與下載。
- 需要支援「全部依序下載」。
- v1 不打包 ZIP。
- 下載多檔時需提醒瀏覽器可能出現多重下載權限提示。
- 下載後要盡快釋放 Blob 資源，但保留歷程摘要。

### 10. 歷程與紀錄

- 每個成功檔案都要記錄歷程。
- 歷程需關聯 `batchId`，方便回看同批次結果。
- 歷程需區分成功、失敗、取消。
- 歷程只保存摘要，不保存媒體檔案。
- 需要顯示批次參數與單檔覆寫摘要，便於重現。
- 成功歷程保留最近 50 筆。

### 11. 狀態與操作

- 批次處理中，使用者可以暫停目前檔案後停止後續。
- 可取消目前檔案並繼續下一個。
- 可取消剩餘待處理項目。
- 失敗不應中斷整批，應續跑後續項目。
- 允許重新處理失敗項目。

## 預設行為

- 來源若已符合目標比例、解析度與格式，預設標記為符合條件並跳過重編。
- 若使用者明確選擇「強制重新輸出」，才重新跑流程。
- 批次設定變更只作用於尚未處理項目。
- 重複加入同一檔案時，保留項目並標記重複。

## 非目標範圍 v1

- ZIP 批次打包下載。
- 透明背景輸出。
- 動態追蹤影片主體的裁切焦點。
- GIF 動畫轉碼。
- 影片與照片混合在同一個佇列中處理。
- 已完成檔案因批次設定變更而自動重做。

## 驗收重點

- 影片與照片批次加入、處理、下載流程一致且分流明確。
- 原始比例、裁切填滿、留白三種模式都能正確工作。
- 單檔覆寫不會污染其他檔案。
- 來源比例與輸出尺寸計算符合預期，且不會拉伸。
- 歷程能回看當次批次與覆寫摘要。
- 失敗、取消、重試、跳過的狀態都可被正確辨識。
