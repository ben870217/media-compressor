# ADR-0036：以獨立「版本與下載」畫面呈現版本目錄

## 狀態

已接受

## 背景

GitHub Pages 必須提供正式版本、EXE 下載與指定版本 PWA 安裝入口，但壓縮工具是使用者進站後的主要工作。將大量下載選項直接置於工作區會分散一般使用者注意力。

## 決策

主壓縮頁的頂端導覽列提供「版本與下載」入口，開啟獨立的版本目錄畫面。壓縮工作區維持預設首頁。版本目錄集中呈現 `releases.json` 的版本卡片、下載資產、checksum、Release notes 與「開啟並安裝 PWA」。

## 後果

- 一般使用者可直接進入壓縮流程；需要指定版本時可明確切換至版本目錄。
- 前端需要在不破壞版本化 PWA scope 的條件下提供此視圖，並支援回到壓縮工作區。
- 版本目錄必須具備鍵盤可操作性、行動版 RWD 與資料載入／無資料錯誤狀態。
