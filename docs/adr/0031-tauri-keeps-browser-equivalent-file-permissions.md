# ADR-0031：Tauri 首發維持瀏覽器等級的檔案權限

## 狀態

已接受

## 背景

應用程式目前使用瀏覽器檔案選擇器讀取使用者明確選取的檔案，並以瀏覽器下載流程交付輸出。Tauri 可額外啟用原生檔案系統或 shell API，但此類權限會增加桌面版與 PWA 的行為差異和安全面。

## 決策

Tauri 首發只使用與瀏覽器相同的檔案選擇器及下載流程；不啟用 Tauri 原生檔案系統、shell、任意路徑存取或其他非必要權限。日後若需要選擇資料夾、批次輸出至指定位置等能力，必須另以最小權限作獨立 ADR。

## 後果

- PWA 與 Tauri 可共用相同的媒體處理與檔案互動邏輯。
- 桌面應用程式不會在使用者未明確選擇檔案時讀寫本機路徑。
- 首發少了原生檔案系統便利功能，但安全審查、測試與跨平台維護較簡單。
