# ADR-0029：固定版本 PWA 可並存，使用者自行升級與回退

## 狀態

已接受

## 背景

版本化 PWA 的目的在於讓使用者可選擇特定 release，且已安裝版本不受 `main` 或後續 release 更新影響。若安裝新版時覆蓋舊版，使用者就失去驗證新版前保留已知可用版本的能力。

## 決策

每個正式版本的 PWA 使用獨立的 manifest identity、start URL、service-worker scope 與 cache namespace。因此不同版本可同時安裝。使用者安裝新版本後自行決定何時移除舊版本；應用程式不自動升級、覆蓋或移除其他版本。

## 後果

- 使用者可保留已知可用版本作為回退，符合指定版本固定的需求。
- 作業系統的應用程式清單可能同時出現多個版本，名稱與圖示應顯示版本號以便辨識。
- 每個正式 release 的 GitHub Pages 資產、manifest 與 cache 均需長期保留且不可覆寫。
