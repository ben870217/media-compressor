# ADR-0030：版本化 PWA 預快取應用程式資產，不快取媒體

## 狀態

已接受

## 背景

版本化 PWA 必須在首次成功載入後，能於無網路的環境啟動與處理使用者裝置上的檔案。但影片與圖片可能很大，且媒體內容屬使用者資料，不應由應用程式的快取策略保存。

## 決策

每個版本的 service worker 預快取該版本的應用程式 shell：HTML、JavaScript、CSS、Web Worker、字型與圖示等建置資產。它不快取使用者選取的來源媒體、壓縮輸出、Blob URL 或預覽資料；這些資料僅在目前執行階段需要時存在。

## 後果

- 使用者首次以 HTTPS 開啟版本化 PWA 並完成快取後，即可離線重新開啟並壓縮本機檔案。
- 每次 release 都有獨立的固定快取，不會讀取或更新其他版本的快取。
- 快取容量可預測且僅受應用程式 bundle 大小影響，不會因使用者媒體而膨脹。
- 快取失敗或儲存空間不足時，介面需清楚告知使用者離線功能尚未可用。
