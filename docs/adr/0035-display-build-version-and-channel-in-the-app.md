# ADR-0035：在應用程式中顯示建置版本與發布頻道

## 狀態

已接受

## 背景

使用者必須知道目前正在使用哪一個版本，尤其 PWA 可同時安裝多個固定版本，且 Pages 根目錄還有會持續更新的 `main` 預覽版。手動寫死版本字串容易和 `package.json`、Git tag 或 Tauri 產物不一致。

## 決策

建置時由唯一版本來源注入應用程式版本與頻道：

- 正式 tag 建置顯示 `vX.Y.Z` 與「正式版」。
- `main` 建置顯示其 package 版本與「預覽版」。

頂端導覽列顯示精簡的版本徽章。Tauri、版本化 PWA 與 Pages 預覽版都使用同一建置版本資料，不在元件內手動維護版本字串。

## 後果

- 使用者可辨識目前執行的是哪個 PWA、桌面版或預覽版。
- Release workflow、Pages workflow 與 Tauri build 必須一致傳入版本／頻道環境值。
- 若注入值缺失，建置必須失敗或顯示明確的開發版標記，不得冒充正式版本。
