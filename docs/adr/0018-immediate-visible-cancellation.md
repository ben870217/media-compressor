# ADR-0018：取消目前任務時立即呈現狀態並中斷處理

## 狀態

已接受

## 背景

原本「取消目前並繼續」只設定旗標，直到轉碼 Promise 結束才更新介面，使用者無法確認取消是否生效。

## 決策

- 使用者按下取消後，任務立即呈現「取消中」，取消按鈕不可重複操作。
- 影片轉碼使用 Mediabunny 的 `conversion.cancel()` 中斷。
- 圖片轉碼終止並重建目前的 Worker 以中斷工作。
- 實際中斷後，任務標示「已取消」並提示將繼續下一項。
- 若工作已進入不可取消的完成階段，保留結果並標示「已完成，取消未生效」。

## 後果

- 取消請求有立即且可理解的回饋。
- 圖片 Worker 的請求管理必須能辨識取消並避免舊 Worker 的結果回寫。
- 使用者可區分「取消請求已送出」、「已取消」及「已完成」三種狀態。
